# Обработка событий через паттерн наблюдателя

В первую очередь это частичная реализация полноценного бота (https://t.me/talking_piska_bot).
Которая хорошо показывает как структурировать проект, с чего начать и как обрабатывать события связанные с пользователями.

# Паттерн Observer (Наблюдатель)

Поведенческий шаблон проектирования, предназначенный для организации односторонней зависимости между объектами: когда один объект (Subject) изменяет своё состояние, все зависимые от него объекты (Observers) автоматически получают уведомление и реагируют на это.

В этом примере паттерн применяется для обработки событий связанных с пользователями и самим ботом находящимся в чате.

## Запуск
  1. Необходимо установить зависимости библиотеки
```bash
bash tnt-tg-bot.pre-build.sh
```
  2. Устновить зависимости примера
```bash
bash app.pre-build.sh
```
  3. Выполнить запуск
```bash
BOT_TOKEN="BOT TOKEN HERE" ./scripts/start --verbose
```

В текущем примере реализовано две приватных команды:
  - /start
  - /help

И одна публичная:
  - /dossier или досье
